How to build a personal blog? (basic)

What is needed to build a blog?

Let’s take this process apart.

Obviousely, a blog is an article on a web page. Everybody know how to write an article in an editor, so the first question is How to make an plan text into a web page?

After doing that, we know a blog can be accessed from anywhere around the world. To achieve this goal, we need a server that provides access service for anybody’s access to your blog. So there are two questions in fact. One is Where to find a server to host my blog? and the other is How to put my blog web pages onto that server?.

Let’s review things above. Basically, the whole build-personal-blog comprises two sub-processes, and we need two kinds of support to help us in each sub-process. Before each sub-process, we need to prepare some much easy things.

preparing articles is easy as long as you can tape in an editor, and I will help you where to find a free server. Now let’s focus on the two sub-process.

Take one to illustrate others

The two sub-processes are universial. Whereas the tools that can provide the two kinds of support are many. We now use two, maybe the most easy-to-use, tools to illustrate how to implement the two sub-processes. First, I will introduce Hexo for the first sub-process. Second, I will use Github as the server to host my blog.

Hexo as web generator

I’d like to write a step-by-step tutorial. But I found that after the tools had been updated, the tutorial may be outdated and misleading. And another con is, an all-in-one tutorial can not improve the reader’s ability to read the original information to solve their own problems. So this tutorial is a guide to lead you to the right place to read rather than copy-paste the original information to my blog.

Now, about Hexo, the offical site provide you a clear docs to lead you, just read it.

Note: There are two things I have to mention.

  1. Do not read all of things. Hexo tool in fact can not only provide text to web support, it can also deploy your web pages onto server. So at current phase, you should only read

    1. Getting Started / Overview. What is Hexo and how to install it.
    2. Getting Started / Setup. Setup a Hexo blog project.
    3. Basic Usage / Writing. How to prepare a “paper” to write using Hexo. The paper is in “.md” format. Do not worry about it now, just edit it like a notepad.
    4. Basic Usage / Generating. How to convert your article into web page.

    No more right now. Too much information will confuse you. That’s why I write this guide.

  2. Why my page can be accessed in browser when I have not picked a server?
    when you run

and you can access a default page in your browser, this in fact makes your PC as a server. This server is running locally and no one except yourself can access it. This is used for test locally, so don’t confuse with the server I menstioned above, which is a remote server and can serve everybody.

Github as web host.

Github is a programmers’ community. They make their projects on it, each project is called a repository. There are two kinds of repositories, one is public, another is private. a public repository can be accessed by other people and can be copied(in Github, called forked) or followed(in Github, called watched) easily. The work thereby can be spreaded among Github. That is a main reason for Github becomming programmers’s community.

Github provide a pages service. It is well understood that a well built project may need a web page to show itself and provide necessary information for its users. It is this service that can be used to build our own blog.

First, you need an account of github. Then you need to create a repository with a dedicated name. read this Step 1. And just step 1. That is enough, do not read the other steps which focus on deployment. Because we will use Hexo to do the deployment.

Connect the two sub-processes

Now you should know how to write and generate your blog web page with Hexo and you have had a Github account. Now we connect github account to our Hexo so we can deploy the blog onto Github.

To do this, read the Hexo Basic / Deployment , especially the Git part.

After you doing that, you can type http://your-github-account-name.github.io in your browser. Your blog will show up, and everybody can access your blog now.
